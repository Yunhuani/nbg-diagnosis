给 Codex 的项目说明书 · NBG 增长诊断产品

这是项目的总图纸。开始任何开发任务前，请先完整读这份说明书，以及随附的四份标准文档。
你的角色：一位资深全栈工程师，按图施工。涉及业务判断（报告长什么样、诊断逻辑）一律以四份标准文档为准，不要自行发挥。


一、这个产品是什么
一个面向中国中小企业主的「AI 增长诊断」工具型网站。客户提交企业经营信息（表单填写 + 文件上传），系统按「NBG 五维方法论」分析，生成一份专业的诊断报告（PPT 格式）。
当前阶段：内部工具（顾问团队使用，AI 出初稿 + 人工审核）。不对外、暂不涉及合规与支付。
商业逻辑分两阶段交付：先诊断报告（本期重点），客户认可后再给增长方案（后期）。本期只做诊断。

二、随附的标准文档（开发的业务依据，必须遵守）

《NBG 诊断报告制作规范》 —— 报告长什么样：视觉（黑金配色）、语言、结构、评分页、标题规则。报告生成模块严格按它。
《采集输入需求清单》 —— 要向客户收集哪些信息、哪些必填（基础项）哪些可选（Plus 项）、缺失时如何降级。采集模块与 schema 按它。
《NBG 产品决策备忘》 —— 技术路线、各模块职责、外部信息可信度要求。架构按它。
《NBG 方案制作规范》 —— 方案阶段标准（本期不实现，供了解全局）。


任何时候，业务问题（"这里该显示什么""这个数怎么算""配色用什么"）先查这四份文档，查不到再问我（指用户）。


三、产品的模块拆解
产品由性质不同的几层组成，分层实现，不要混在一起：
模块职责技术财务计算确定性核算：产品线全成本盈亏、客户集中度、现金跑道、应收占用。必须用代码精确算，不允许调用大模型生成数字。Python，独立模块五维分析编排调大模型 API，按 NBG 五维逐维分析 + 交叉验证，输出结构化结论（JSON）。外部行业信息须用联网检索、带来源，不用模型记忆。Python + 大模型 API（已备案国产模型 / 顶级模型）报告生成把结构化分析结果 + 财务计算结果，套固定模板，渲染成 PPT。Node + pptxgenjs；图表用 matplotlib 出深色透明 PNG 或 PPT 原生图表采集系统表单 + 文件上传，按采集清单 schema 收集数据。全栈建站工具（如 Lovable）或自写前端账号与存储用户账号、提交记录、文件存储。同上 / Supabase 等结果展示展示报告、提供下载。同上

四、施工顺序（严格按此，一次一个模块，跑通再下一个）
核心纪律：内核先行、一次一块、跑通再继续。不要一次性生成整个产品。
阶段一 · 内核（先做，本地跑通，无界面）

财务计算模块 → 输入模拟产品线数据，输出正确的盈亏/集中度/现金数字
五维分析编排 → 输入模拟企业数据，输出五维结构化结论
报告生成模块 → 输入上面两步的结果，输出一份 PPT


阶段一验收：命令行里跑「模拟数据 → 完整报告 PPT」全程通。

阶段二 · 外壳（内核通过后）
4. 采集表单 + 文件上传
5. 账号与数据存储
6. 结果展示与下载
阶段三 · 接合
7. 采集数据 → 喂给内核 → 生成报告 → 展示下载，端到端打通。

五、已沉淀的内核资产（可直接参考/复用，无需从零）
在与 AI 的设计过程中，以下已验证可行，开发时作为参考实现：

财务计算逻辑：产品线全成本 P&L、隐性出血点、客户集中度、现金跑道、应收占用的计算方法（见《采集清单》财务维 + 下方示例数据）。
五维内容逻辑：每维「核心判断 + 论据 + 反转发现」结构，已对齐方法论。
报告模板：黑金配色、action title、体检评分页、各类版式（反转对比、三列表、矩阵、雷达图等）——已有可运行的 pptxgenjs 实现作为范例。
完整样板：一份 18 页诊断报告样板（卫浴五金模拟案例「甬辉」），是产出目标的标准。


上述资产的具体代码，用户会在对应任务时提供给你参考。你的工作是理解它、整理成产品级的、可维护的模块，而非推倒重来。


六、关键红线（违反会砸掉产品信誉，务必遵守）

财务数字必须代码算，不允许大模型生成。
外部行业数据必须联网检索、带可追溯来源，不用模型记忆里的"大概数字"。
报告视觉严格按《诊断报告制作规范》：黑金配色、红粉禁用、图表深色融入不放白卡片、标题是观点结论。
一次一个模块，跑通再下一个，不要图快一次生成全部。
