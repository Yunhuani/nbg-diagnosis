# NBG 咨询产品 · 决策备忘 V1.1

> 本文档记录 NBG 咨询产品在产品形态、技术路线、实现策略上的关键决策与共识。与《NBG 诊断报告制作规范》互补：那份管「报告长什么样」，这份管「产品怎么做」。

---

## 一、产品形态

- **当前定位**：内部使用的工具型网站（顾问/团队用，AI 出初稿 + 人工审核深化）。
- 因为是内部使用，**不涉及对外服务的合规要求**（ICP 备案、生成式 AI 算法备案、微信类目等均不适用）。
- 若未来转为对外（公众可访问、收费、带 AI 生成），需重新评估合规：届时 ICP 备案 + 生成式 AI 算法备案大概率绕不开，这部分建议找专业代办，规则以届时官方最新口径为准。

## 二、交付物分两阶段（商业逻辑）

1. **第一阶段：诊断报告**——先做、更重要。证明价值 + 挑动付费动机。
2. **第二阶段：增长方案**——客户认可诊断后再给（三步方案：增长杠杆筛选 → 三阶段路径 → 90 天路线图）。
- 这样分阶段，每一步都有付费理由，商业逻辑成立。
- 当前聚焦：把诊断报告做到最好。方案阶段后续再做。

## 三、诊断报告：单一版本，自适应输入

- 目前只做**一个诊断版本**，质量优先，不分简版/复杂版（避免复杂化）。
- **自适应输入**：客户给定性信息就给方向性判断；给真实数据（财报、产品线、客户清单）就量化到具体金额。有什么用什么，全部整合进同一份诊断。
- 数据越真，发现越锋利（如隐性出血点必须有真实产品线数据才算得出）。

## 四、技术路线（自己用 AI 工具搭）

### 4.1 总原则
- 产品由多段性质不同的活组成，**没有单一工具能全包**，分层选型：
  - 承载层（网站界面、表单、上传、账号）
  - AI 大脑（五维分析编排）
  - 确定性计算（财务）
  - 报告生成（PPT/PDF）

### 4.2 各层选型结论
- **内核先行**：先把最值钱、最不可替代的「核」做扎实，再包「壳」。顺序类比 NBG 的「固本→增长→破局」。
- **编写工具**：用 **Codex** 编写内核（已确定）。Codex / Claude Code 在写复杂逻辑上同级，选顺手的即可。
- **AI 编排**：直接用代码调「已备案的国产大模型 API」或顶级模型 API；**Dify 等可视化编排平台非必需**（对一体化、数据干净度反而是负担）。五维编排逻辑 = 一维一次调用 + 最后交叉验证。
- **确定性财务计算**：必须用代码精确算，**不能让大模型自由生成数字**（会砸品牌）。
- **报告生成**：pptxgenjs 出 PPT；图表用 matplotlib 出深色透明 PNG 或 PPT 原生图表。
- **承载层**：内部工具阶段可用全栈 AI 建站工具（Lovable 等）快速搭壳；部署在哪都行、不用备案。

### 4.3 实现顺序
1. 先做内核：五维分析提示库 + 财务计算代码 + 报告生成模板，本地跑通「模拟数据进 → 完整报告出」。
2. 再搭壳：采集表单、上传、账号、结果展示。
3. 把核接进壳。
4. （若对外）最后接支付、合规、上线。

## 五、报告形态 = PPT

- 交付物用 **PPT**（不是 Word/PDF），因为 PPT「一页一观点」、扫读友好、适合给老板看。
- 报告页数指 PPT 页数，约 18–22 页。

## 六、已沉淀的内核资产（在制作环境中已实现，可供 Codex 组装）

- 财务计算模块（确定性核算：产品线全成本 P&L、隐性出血点、客户集中度、现金跑道、应收占用）。
- 五维内容逻辑（每维核心判断 + 论据 + 反转发现，已对齐方法论）。
- 图表生成（深色透明：雷达、价值曲线、波特五力、产品线、集中度、能力矩阵、新兴市场增长）。
- PPT 模板代码（pptxgenjs，黑白金配色 + action title + 各类版式组件：反转对比、三段论据、金条结论、维度封面等）。
- 完整 18 页诊断报告样板（卫浴五金出口模拟案例）。

## 七、外部信息的获取与可信度（关键）

> 报告除客户内部信息外，还需大量外部信息：市场研究、宏观分析、竞争对手。可信度是产品的信任根基。核心原则：**不靠"声称它真"，靠"让客户能自己验证"。**

### 7.1 信息分层取数（按可信度）
- **一手权威源（优先）**：政府/海关统计、行业协会、上市公司财报与招股书、知名调研机构公开报告（QYResearch、灼识、艾瑞等）、大平台官方行业数据。
- **可信二手源**：主流财经/垂直媒体深度报道，需能追溯到其引用的一手数据。
- **辅助源（仅作线索，不作论据）**：论坛、自媒体、问答——帮发现行业痛点，但必须再用一手源验证，不可直接引用。
- **红线：禁止使用模型记忆里的"大概数字"。** 外部信息必须来自实时检索，不能模型脑补（会过时或编造，砸品牌）。

### 7.2 让客户信服的四个手法（须落进产品）
1. **逢数据必带来源（可追溯）**：每个外部数字标出处，客户能自己核。带不出可信来源的数字宁可不写。这是区别于"AI 裸聊黑箱结论"的关键。
2. **技术上用"带检索的 AI"，不是"凭记忆的 AI"**：Codex 开发时，外部信息模块必须接联网检索（先搜真实网页、基于来源写、强制标注引用），从机制上杜绝编造。
3. **区分"硬数据"与"判断"**：事实（市场规模、增速、对手营收）必须带源可核；判断（如窗口期）要讲清推理依据。两者视觉上区分，信任更扎实。
4. **附"信息来源清单"页**：报告/方案末尾列出所有引用的权威源，本身就是"做了真实案头研究"的证明。

### 7.3 竞争对手信息（最难，特别小心）
- **能查到的**（上市对手财报、公开的产品/价格/渠道）→ 带源用。
- **查不到的**（私人公司内部数据）→ **不要编**，改用"行业典型水平/可观察的市场行为"推断，并说明是推断。
- 宁可承认"只能基于公开信息推断"，也不给看似精确实则编造的对手数字——后者被识破则整份报告信任崩塌。
- 与"财务用代码算、不让模型生成数字"同一精神：**凡客户会拿来做决策的数字，要么可计算、要么可溯源，绝不能模型脑补。**
