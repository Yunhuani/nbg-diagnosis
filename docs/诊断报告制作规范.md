# 云华 NBG 增长诊断报告 · 制作规范 V1.0

> 本规范是「诊断报告」这一交付物的统一标准。无论由谁制作、用什么工具，都应遵循以下规则，以保证每一份报告的专业性、品牌一致性和价值感。
> 适用对象：诊断报告（五维诊断）。方案阶段（三步方案）为后续独立交付物，另行规范。

---

## 一、核心定位

诊断报告要同时做到两件事：

1. **证明价值**——让客户感到「他们比我自己还看得清我的生意」（建立信任）。
2. **挑动动机**——让客户产生「那我到底该怎么办」的焦虑（引出付费的方案阶段）。

一句话原则：**对「发现」要慷慨，对「处方」要克制。**
诊断可以毫无保留地指出问题有多严重、为什么；但「具体怎么做」是方案阶段的内容，诊断只点到「方向」，不给「路径/操作」。

> 守住这条线，诊断才既有料、又给方案留出空间。

## 二、视觉规范

### 2.1 底色与配色体系（黑 + 白 + 金，绿点缀）

- **底色**：冷炭黑 `#121317`（封面/收口可用更深的 `#0E0F13`）。不用纯黑（显廉价），不用深蓝以外的杂色。
- **主体文字**：白 `#EAEDF3`；次要文字浅灰 `#9DA4B3`；最弱信息 `#6E7180`。
- **唯一高光色**：金 `#E8B84B` / 亮金 `#FFD166`。用于标题关键词、关键数字、点睛结论、序号、强调元素。**金色是全报告唯一的高光色，贯穿所有页。**
- **绿色** `#6FCB94`：作为**第二功能色**，承载「正向/机会/改善/增长」语义——如预期效果、增长动作、健康评分、上升箭头。可比早期更主动地使用，但仍以黑金为主、绿色为辅，避免喧宾夺主。
- **橙/琥珀** `#E0A05A`：作为「警示/亚健康/止血」语义的中间色（介于金与暗灰之间），用于评分条、问题侧标识。比红色克制，符合咨询风。
- **红 / 粉：禁用**。咨询风格不用红粉。
- **表达「负面 / 风险」的方式**：不用红，改用「暗灰下沉」——框体 `#1A1B20`、描边 `#3A3C44`、文字 `#8A8F9C`，视觉上自然「弱下去、沉下去」。
- **表达「正面 / 机会」的方式**：金色高亮框 `#231F16` + 金色描边 + 白字，视觉上「亮起来」。
- 核心手法：**用明暗对比（暗沉 vs 高亮）表达好坏，而不是用红绿对比。** 这是咨询级的克制做法。

### 2.2 图表规范

- 图表必须是**深色透明背景、直接融入黑底**（线条、文字用白/金/灰）。
- **禁止把图表放在大白卡片里**——黑底上糊白板显廉价、不融合。
- 负值/负面数据用暗金或暗橙 `#C77B3A`，不用红。
- 原生可编辑图表优先（PPT 的 RADAR/LINE/BAR）；复杂图（价值曲线带标注等）可用 matplotlib 出透明 PNG。

### 2.3 排版与禁忌

- 版式：16:9 宽屏（LAYOUT_WIDE，13.33×7.5 英寸）。
- **禁止**：标题下划线、装饰性全宽色条、页眉页脚色块——这些是 AI 痕迹。用留白和字号对比拉层次。
- 字号对比要足：标题 30–34pt、维度大标题 50pt、正文 13–15pt、辅助 10–11pt。
- 留白充分，底部内容与页脚之间务必留出间距（避免重叠）。
- 中文字体统一（思源/微软雅黑系），不混用。

## 三、语言规范

### 3.1 专业但不口语

- **去口语化，但不去棱角**。判断要鲜明有力，但用书面的、概念化的语言承载，不用「红海、卷、握牌、唠」这类自媒体腔。
- 错误示例（太口语）：「你在一片红海里和所有人拼价格」
- 正确示例（专业有力）：「你把最大的力气，花在了最不赚钱的战场上」

### 3.2 有代入感（客户视角，不是框架视角）

- 把抽象命题翻译成**客户自己的处境和后果**，让他一读就知道「这在说我」。
- 错误（框架视角，冰冷）：「竞争结构判定：正面价格竞争不具备可持续性」
- 正确（客户视角，有代入感）：「你越靠压价守单，毛利就越薄——这条路走不通」
- 用第二人称「你」，锚定在客户真实经验上，而非分析框架上。

## 四、结构规范

### 4.1 标题即观点结论（McKinsey Action Title）—— 最重要的一条

- 每一页顶部的标题，必须是**这一页的观点结论本身**，不是主题标签。
- 客户光读标题（不看内容）就能 get 这页的核心论点。所有页标题串起来 = 整份报告的论证主线。
- 错误（主题标签）：「最关键的发现」
- 正确（观点结论）：「你的『大客户依赖』，是壁垒，不是软肋」
- 标题控制在一行（约 15–25 字），关键词用金色强调。
- **好懂优先于精炼**：标题宁可长一点、说人话，也不要精炼到客户要想一下才懂。给老板看的东西，第一眼读懂 > 文字漂亮。
  - 过度精炼（难懂）：「堵住正在漏的」
  - 好懂（推荐）：「第一步：先把每年悄悄亏掉的钱，堵住」

### 4.2 一维一个核心判断

- 每个维度收敛成**一个核心判断**，不要并列多个诊断。多个诊断会让客户难以抓重点。
- 维度内的各页是支撑这一个核心判断的分论点/论据。
- 五维 = 五个核心判断，客户要记住的就五句话。

### 4.3 五维之间要交叉呼应

- 同一个事实从不同维度交叉照射，形成全局洞察（这是单点问 AI 给不出的）。
- 范例：客户集中度在「财务维」是风险，在「竞争维」是资产——同一事实，两个角度。

### 4.4 每页一个观点（PPT 逻辑）

- PPT 一页 = 一个观点 + 支撑视觉。信息量克制、留白充分、扫读友好。
- 宁可一个观点单独成页（20+ 页每页清爽），也不要一页塞两三个观点（12 页每页拥挤）。给老板看，扫读 > 阅读。

### 4.5 体检式评分页（像验血报告一样醒目）—— 诊断报告必备

- 诊断报告靠前位置必须有一页「企业增长健康度体检表」，像体检/验血报告一样让客户一眼看清现状：
  1. **一个醒目的总分**：五维等权平均，大号字呈现（如 5.0 / 10），配一个判定标签（健康 8-10 / 亚健康 5-7 / 警告 1-4）。
  2. **五维分项评分条**：每维一个分数 + 进度条 + 一句话状态，按分数高低排列，用绿（健康）→金→橙（警示）的颜色梯度。
  3. **一段具体的体检结论**：不能是「亚健康/需干预」这种笼统标签，必须是点出该企业真实结构性问题的一段话，且与五维分数对应。
     - 错误（笼统）：「综合 5.0 分，亚健康，需重点干预。」
     - 正确（具体）：「不是不行，而是『外强中干』——市场竞争尚可（6.5/5.5），越往里的模式、财务越弱（4.5/3.5），最紧迫的是先止住财务失血。」

### 4.6 框架导航（让客户不迷路，解决单页割裂）

- 报告需有「全景框架页」展示整体结构（五维全貌 / 方案的战略主张→四类行动→三阶段），让客户看到各页之间的关系。
- 维度/章节页的页眉处可放一个「定位条」，高亮当前所在板块（类似「你在这里」），无论翻到哪页都知道位置。

## 五、专业性与深度要求

### 5.1 三样「咨询痕迹」（区别于 AI 裸聊）

1. **结论先行**：每页/每块先抛出有棱角的判断，再给支撑（金字塔原理）。
2. **So-What 逼问到底**：把一个数据顺着因果链追问三四层，逼出客户想不到的结论。
3. **量化带参照系**：每个数字配一个基准线（如「65% vs 行业健康线 40%」），证明「我们见过同类企业」。

### 5.2 框架双署名

- **NBG 五维是骨架**（品牌资产），**每维一把经典咨询框架是手术刀**：
  - 市场与机会 → MECE 机会拆解 + 波特行业分析
  - 竞争格局 → 波特五力 + 蓝海战略客户价值曲线
  - 商业模式 → 商业模式画布
  - 内部能力 → 能力-资源矩阵
  - 财务健康度 → 杜邦分析 + 全成本作业法 + 营运资金周期模型
- 每页底部标注「分析框架：XX」+「泽思 AI 按 NBG 五维框架生成 · 经顾问审核深化」。

### 5.3 真实数据，不可编造

- 行业分析（市场规模、增速、竞争格局、标杆案例）必须来自**真实可核查的检索**，带来源，不用模型记忆里的泛泛之谈。
- 财务计算（隐性出血点、客户集中度、现金跑道等）必须用**代码精确核算**，不可由模型自由生成数字。
- 这两点是「不像 AI 裸聊」最实的来源。

### 5.4 每维至少一个「客户想不到」的发现

- 若一维的结论客户全都已知，这一维就没价值。
- 最有杀伤力的是「反转认知」型发现（如：你以为的风险其实是资产）。

## 六、标准页面清单（约 18–22 页）

1. 封面（同心环意象 + 诊断对象 + 泽思品牌 + 日期 + 保密）
2. 目录（五维一览）
3. 我们怎么做诊断（方法论 + 差异化对比 + 模型差异点）
4. 企业概况（基本盘事实）
5–6. 维度一 市场与机会（封面 + 论据页）
7–8. 维度二 竞争格局（封面 + 核心发现页）
9. 维度三 商业模式（封面/画布页）
10. 维度四 内部能力（封面/矩阵页）
11–12. 维度五 财务健康度（封面 + 隐性出血 + 现金安全）
13. 五维雷达总览（评分 + 短板/机会判断）
14. 三个关键发现汇总（会被截图转发的一页）
15. 收口（最大风险 + 最大机会方向，过渡到方案阶段）

> 维度页数按复杂度伸缩：市场/竞争/财务可 2 页，商业模式/内部能力可 1 页。

## 七、模型差异点的正确用法

- 客户多用免费模型（豆包等），我方用 Claude、GPT 等顶级付费模型——这是真实差异点，可以突出。
- **但用「稀缺/门槛」陈述，不用「优越」自夸**：强调「中国境内多数企业用不到」，而非「我们更高级」（前者是客户认的事实，后者会被反问）。
- **突出模型后必须立刻锁上护城河**：紧跟一句「模型只是起点，真正的价值在于用 NBG 方法论驾驭它、再由顾问审核深化」——防止客户产生「那我自己充个会员就行」的念头。
- 位置：放在「我们怎么做诊断」页，作为差异化的一部分，不单独成页吹模型。

## 八、技术实现备忘（制作工具）

- PPT 生成：`pptxgenjs`（Node）。
- 图表：matplotlib 出深色透明 PNG（价值曲线/矩阵等带标注的），或 PPT 原生图表（雷达/折线/柱状）。
- 财务计算：独立 Python 模块，确定性核算，数字喂给报告。
- 渲染检查：soffice 转 PDF → pdftoppm 转图 → 视觉检查重叠/溢出。
- 配色、字号、间距的具体数值见本规范第二节。
