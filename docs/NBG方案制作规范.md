# NBG 方案制作规范

## 一、适用位置

方案阶段发生在诊断之后、客户认可诊断结论之后，作为独立交付物输出。

## 二、方案五层结构

1. **战略主张（strategic_thesis）**
   一句话的「从 X 转向 Y」方向性抉择。必须有取舍，明确做什么、不做什么，并扣住诊断结论。

2. **杠杆选择矩阵（lever_matrix）**
   从战略主张推出候选增长杠杆，按「影响力 × 可行性」评估排序，聚焦选出少数几个优先发力杠杆。

3. **四类行动地图（action_map）**
   把选中杠杆拆成具体行动，并按四类行动归位。

4. **三阶段路线图（roadmap）**
   把行动地图中的行动按时间先后和真实依赖排入三个阶段。阶段名称按本案例战略节奏确定，每阶段明确目标、行动、排序理由和里程碑。

5. **90 天行动计划（ninety_day_plan）**
   把三阶段路线图的第一阶段行动细化为未来 90 天内可直接执行的任务，明确负责人、时间节点、产出物、衡量标准和真实上游依据。

## 三、四类行动地图定义

四类行动均按领域维度划分：

- **战略方向**：关乎「往哪走」的方向性动作。
- **战术方向**：产品、业务、渠道、运营层面的打法动作。
- **管理方向**：组织、人才、流程层面的动作。
- **风险财务**：守住底线、保障现金、防范风险的动作。

## 四、方案各层质量原则

代码校验只守客观底线：

- 字段完整。
- 枚举合法。
- `grounded_in` 引用真实存在的诊断结论或上层结论。
- 防幻读，例如产品线盈亏必须符合代码计算事实。
- 防串台，例如不得引用本案例不存在的行业、市场、渠道或机会方向。

具体性、专业性等主观质量，不由代码硬卡。它们由提示词要求模型保证，并由顾问审核把关。
